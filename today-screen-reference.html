Today Screen Tiles

Every tile that can appear on the Today screen — what data it reads, when it shows, how it works, and where it navigates. Cards are assembled dynamically at render time and sorted by a fixed priority order.

How the Today screen works

Each render call builds a fresh cards[] array, sorts it, then writes the result into #today-content. Nothing is cached between renders.

Render trigger
renderToday() is called whenever the user taps the Today tab, or when any card action (e.g. dismissing setup) triggers it directly.
All state is read from the global state object (persisted to Firestore + localStorage via saveData(state)).
Cards are conditionally included — if the condition returns no data the card is simply omitted from the array.
Day-part detection
overnight — 12am–4:59am
morning — 5am–11:59am
afternoon — 12pm–4:59pm
evening — 5pm–8:59pm
night — 9pm–11:59pm
Evening and night shift the Schedule card and greeting to show tomorrow's data instead of today's.
G
Always present — not a card

Greeting + Briefing

Rendered above the cards area on every load. Not included in the sortable cards array.

THURSDAY, 1 MAY
Good morning, Robert.
Electricity bill is due today.
Data sources
Name — resolves from Firebase _currentUser.displayName, then state.settings.adultName, then state.settings.adults[0].name, then state.householdProfile.members (first adult).
Date line — current device date, formatted THURSDAY, 1 MAY via toLocaleDateString('en-AU').
Greeting word — time-aware: Good morning / Good afternoon / Wind down / Tomorrow at a glance / Still up.
Briefing line — single sentence generated from card state: mentions upcoming bill if overdue, or number of calendar events if ≥3, otherwise shows a calm message like "Quiet day ahead."

Setup Progress

Guides new users through configuring their household. Dismisses permanently once the user taps "Dismiss" or all tasks are complete.

Finish setting up Toto
2 of 6 complete
33%
Conditions & behaviour
Shown when — state.setupProgressDismissed is falsy AND at least one setup task is incomplete.
Tasks tracked — household profile, first adult, first child (if applicable), first bill, first calendar event, first budget entry. Derived by setupProgressTasks().
Collapsed by default — shows headline + circular progress + progress bar only. Tap to expand the full task list.
Completed state — when all tasks done, replaces the card with a confetti card that the user can dismiss.
Dismiss — sets state.setupProgressDismissed = true and saves. Card never reappears.

The single highest-urgency item. Only one priority card renders at a time, in this order: bills due ≤2 days → maintenance overdue → documents expired.

Data sources
state.bills[] — filtered to bills where billDaysUntil(b) ≤ 2 and not null. Sorted by days ascending.
state.maintenance[] — filtered where _maintDaysUntil(m) < 0 (past due).
state.documents[] — filtered where expiryDate < now.
Display rules
Bills trigger first. If any bills are due within 2 days, the card shows. Maintenance and document expiry are only shown if there are no upcoming bills.
Single bill: shows bill name + amount as headline with "Pay now →" button.
Multiple bills: count in iris pill, first 2 shown in list, "+ N more · Total due $X" footer if there are more than 2, "See all →" link in eyebrow.
Overdue maintenance: ember colour scheme, shows first overdue item, "View →" navigates to Maintenance tab.
Expired documents: same iris style, shows document name + "View →" to Documents tab.
Navigation
"Pay now →" → activateTab('bills')
"View →" (maintenance) → activateTab('maintenance')
"View →" (documents) → activateTab('documents')

Money

Current month's budget snapshot — how much is left (or over) and what percentage of the budget has been spent.

On track
May 2026
$840 left
On track
Electricity due today 23 days left
Over budget
May 2026
$215 over
Over budget
23 days left
Data sources
getMonthData(selectedBudgetMonth) — returns income + expenses arrays for the selected month.
monthlyTotal(income) / monthlyTotal(expenses) — converts all frequencies to monthly equivalents and sums.
getActual(e.id, month) — sum of real transactions recorded for each expense category.
Bill flags are pulled from the same billsDue array used by the priority card, showing bills due today/tomorrow/in N days.
Display rules
Only shown if at least one income or expense entry exists (totalIncome > 0 || totalExpenses > 0).
On track — surplus ≥ 0. Shows amount in green-tinted pill.
Over budget — surplus < 0. Pill switches to ember, suffix changes to "over", bar fills amber.
Progress bar = totalActual / totalExpenses * 100, capped at 100%.
"N days left" flag always shown. Bill warning flags shown for any bills due within 2 days.
Navigation
Tapping anywhere on the card → activateTab('budget')

Conditional · kids (type)

My Routines

The logged-in user's personal routines that are active right now or within the next 90 minutes. Inline step checklist with progress bar.

My Routines1
🌅
Morning routine 7:00
1/3
✓
Make bed 2m
Shower 10m
Pack bag 5m
Data sources
_routinesForCurrentUser() — routines assigned to the logged-in user, filtered by _routineMatchesDate(r, todayStr) (day-of-week or date match).
r.completions[todayKey][] — array of completed step IDs for today's date key (format YYYYMMDD).
r.triggerTime — HH:MM string. If absent, routine is always visible.
Time window filter
A routine is visible if current time ≥ triggerTime − 90 min AND < triggerTime + 360 min (6 hours).
A routine that has already been started today (has any completions) is always shown even if outside the time window.
Routines outside the window show a locked state with no step list and an availability label.
Interactions
Tapping a step calls _tdToggleStep(routineId, stepId) — toggles completion in r.completions[todayKey] and re-renders.
When all steps complete, card turns green with "✓ Done" label.
If any kids have pending approvals, a "N approvals waiting →" button appears below routines, navigating to activateTab('kids').

Navigation
Tapping the card → navigates to the relevant section (Documents / Maintenance / Vehicles) depending on the first item type.

Coming Up

Planner events and bills due in the next 3–7 days. A forward-looking nudge so nothing sneaks up on you.

Coming up3
Sat 3 May Soccer carnival
In 4d Phone bill $45
Sun 7 May Mother's Day lunch
Data sources
state.planner.events[] — filtered to event.date > today AND ≤ today + 7 days, sorted by date, capped at 3.
state.bills[] — filtered to bills where billDaysUntil(b) > 2 AND ≤ 7.
Display rules
Only shown if at least one upcoming event or bill exists in the 3–7 day window.
Events and bills are combined and capped at 4 rows total.
Events show the formatted date (e.g. "Sat 3 May") and event title. Bills show "In Nd" and bill name + amount in muted mono.
If an event has a time, it's shown below the date in iris mono.
Navigation
Tapping an event row → activateTab('planner') + opens event detail via _plannerOpenDetail(ev.id)
Tapping a bill row → activateTab('bills')

Card sort order

Cards are sorted by a fixed typeOrder map. Cards of the same type would sort by insertion order (though each type produces at most one card).

Position Card type Title shown Condition to appear Navigates to
G Greeting — (always present, above cards) Always rendered n/a
0* Setup Progress Finish setting up Toto Not dismissed + at least 1 task incomplete Relevant settings section per task
1 priority Heads Up / Overdue / Expired Bills ≤2 days, OR maintenance overdue, OR docs expired Bills / Maintenance / Documents
2 schedule Today / Tomorrow At least 1 planner event for today (or tomorrow if evening/night) Planner tab; event detail sheet
3 money [Month Year] At least 1 income or expense entry in budget Budget tab
4 lists (routines) My Routines At least 1 routine visible in current time window Kids tab (for approvals)
5 slipping Slipping At least 1 expired doc, overdue maintenance, or expired rego Documents / Maintenance / Vehicles
6 upcoming Coming up At least 1 planner event or bill in the 3–7 day window Planner (events); Bills (bills)
7 win Win At least 1 child with a routine streak ≥ 3 days n/a (celebratory only)
8 lists (paired tiles) Shopping List + Kids Always rendered (Kids tile only if child profiles exist) Lists tab (shopping); Kids tab
Notes
* Setup Progress is rendered separately before the sorted cards array and is not part of the typeOrder sort.
The calm state — "You're sorted. See you tomorrow." — appears when the cards array has 1 or fewer items.
An async AI briefing (_fetchAIBriefing()) runs after the DOM is written and may update the briefing line with a personalised summary.
All card renders are wrapped in safeRender() — errors are silently swallowed so a broken card never breaks the whole Today screen.
