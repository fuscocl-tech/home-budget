How to use this prototype

This is a fully interactive calendar prototype to test the design before it goes into the app. All data is stored in memory — nothing is saved between refreshes.

  • Month/Week toggle — switches calendar view
  • ‹ › — navigate months
  • + button — add a new event
  • Tap a day — scrolls agenda to that date
  • Tap an event — opens edit modal
  • Tap a member chip — filters calendar to that person
  • Life Areas / Heads up — collapsible sections
Design decisions in this build:
• Dots in grid = per member, not per event
• Agenda always visible below the grid
• "Who" is first field in modal
• Save button colour matches selected member
• Life areas = compact chip strip, not large tiles
• Tapping a day scrolls agenda (no page change)
