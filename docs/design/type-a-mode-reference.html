Feature Reference · Type A Mode

Type A Mode

An opt-in overlay that transforms TOTO from a passive dashboard into an active life coach. When enabled, it adds a Life Score, daily missions, and a weekly reset ritual — all designed to guide the user toward a more organised household one step at a time.

What it is

Type A Mode is a user-controlled mode toggled in Settings. It does not affect any existing features — it only adds to them.

Mode off (default)
Today screen shows reactive cards only — what's happening now, what's due, what's slipping.
No Life Score, no missions, no weekly nudge.
User drives their own organisation rhythm.
Mode on
Life Score — a 0–100% health metric across 5 life dimensions, always visible on the Today screen.
Daily Mission — one specific, high-impact action for the user to take today, targeted at their weakest area.
Weekly Reset — a 5-step guided ritual to review and plan for the week ahead.
Streak — tracks consecutive weeks the user completes their weekly reset.
How to enable
Settings → Type A Mode accordion → toggle Enable Type A Mode. Calls updateSetting('typeAMode', true) which sets state.settings.typeAMode = true and persists to Firestore.
The Settings panel shows the current Life Score percentage immediately after enabling so the user gets instant feedback.

Life Score

A single 0–100% number that represents how organised the household is right now. Calculated fresh on every render from live state — never stored.

Score bands
80–100% — "Crushing it — keep going"
60–79% — "Good shape — a few things to tidy"
40–59% — "Getting there — some gaps to fill"
0–39% — "Just getting started — let's build momentum"
How it's calculated
Five dimensions each scored 0–100 independently via calcLifeScore().
Total = simple average of all five dimension scores: Math.round(sum / 5).
Recalculated on every renderToday() call — always reflects current state.
The score is never stored in state. It's a derived value, not a persisted one.
Ring colour: green ≥80, cyan ≥60, amber ≥40, red below 40.
The same score is also displayed in the Settings panel for quick reference without navigating to Today.

Daily Missions

Each day, Type A mode surfaces one specific, actionable task targeted at the user's weakest dimension. It's always the highest-impact thing they could do right now.

⚡ Today's Mission
Log this month's actual spending
Import a bank CSV or add manually
+30 pts to Life Score · Budget dimension
Mission escalation
Day 1 — Mission card shown on Today screen. No pressure.
Day 2+ — A modal lightbox appears 1.5 seconds after app load showing how many days the mission has been ignored.
User can always tap Skip to dismiss for today. The dismissed mission ID is stored so a new mission is shown tomorrow.
How missions are selected
All 5 dimensions are scored and sorted weakest-first.
Each dimension generates specific candidate missions based on the exact gap (e.g. "no income set up" vs "no actuals logged" are different Budget missions).
Candidates are filtered to remove today's already-dismissed mission.
Sorted by impact score — the mission with the highest potential Life Score gain wins.
If no missions are available (everything in good shape), nothing is shown.

Plan dinners: +25
Mark maintenance done / set up lunchbox: +20
Pantry, shopping list, goals: +15

Weekly Reset

A 5-step guided ritual the user can run once a week to review the past week and prepare for the next. Completing it earns a streak point.

The 5 steps
Step 1 — Budget review: Shows month-to-date spending vs budget. Tap "Review budget" or skip.
Step 2 — Meal plan: Shows how many of next week's 21 meal slots are filled. Tap to go plan.
Step 3 — Pantry check: Shows how many pantry items need restocking. Tap to run stocktake.
Step 4 — Upcoming bills: Lists bills due in the next 7 days with total amount.
Step 5 — Maintenance: Shows overdue maintenance count. Tap to review.
Done: Tap "I'm reset" — streak increments, typeALastResetDate updates.
Behaviour details
Each step can be individually skipped — the card fades to 40% opacity. The user doesn't have to complete every step.
Tapping a step action navigates the user to the relevant tab and closes the reset overlay.
Opened via a "Weekly Reset" button on the Today screen (Type A mode only).
The overlay closes if the user taps outside it.
Data shown in each step is calculated live from state at the moment of opening — always current.

Streak System

Tracks how many consecutive weeks the user has completed their Weekly Reset. Shown as a 🔥 badge on the Life Score card.

How streak is tracked
state.settings.typeAStreak — integer, number of consecutive weeks.
state.settings.typeALastResetDate — ISO date of last completed reset.
Continued: If the last reset was ≤9 days ago, streak increments by 1.
Broken: If last reset was >9 days ago, streak resets to 1 (not 0 — the current reset counts).
9-day window (not 7) gives a 2-day grace period for users who do their reset on a different day each week.
Display
Shown as 🔥 N week streak badge on the Life Score card — only if streak > 1.
Also shown in the Settings panel Type A accordion when Type A Mode is active.
Streak of 1 is not shown — it would feel hollow after just one reset. Starts displaying from week 2.

State & Storage

All Type A settings live under state.settings and are persisted to Firestore via saveData(state).

Keys
typeAMode — boolean. Default: false. Master switch for the entire feature.
typeAStreak — integer. Default: 0. Consecutive weekly reset count.
typeALastResetDate — ISO date string. Default: ''. Date of last completed Weekly Reset.
typeAMissionId — string. Default: ''. ID of the mission currently being tracked.
typeAMissionShownDate — ISO date string. Default: ''. Date the current mission was first shown — used to calculate days ignored for escalation.
typeADismissedMission — string. Default: ''. ID of the mission the user dismissed today — excluded from selection until a new mission is generated.
All keys are initialised in _applyMigrations() so they always exist regardless of when the user first signed up.

How it should evolve

Type A Mode has a strong foundation but several gaps where the experience could be meaningfully improved. These are roughly in priority order.

High priority
Shopping list integration into Family dimension
The Family score currently uses the old meals.shopping and meals.pantry arrays. Now that shopping has moved to state.lists.food, the dimension score should reflect the new system — active items, got-it count, and list completeness. The pantry concept has also merged into the food list so the scoring logic needs updating.
High priority
Routines as a scored dimension
Routines are a significant part of the app but have no representation in the Life Score. Adding a Habits or Routines dimension (e.g. % of today's routines completed, streak of consistent days) would make the score more meaningful for daily users and give Type A mode a daily touchpoint beyond just planning tasks.
Medium priority
Mission completion feedback
Right now, completing a mission just resets the tracking. There's no celebration or score delta shown. A "Life Score went up X points" confirmation — even just a brief toast — would make the action feel rewarding and reinforce the connection between missions and the score.
Medium priority
Score history chart
The score is calculated live but never stored. Periodically snapshotting it (e.g. weekly at reset completion) would allow a simple trend line — "your score has improved from 54% to 72% over the last 8 weeks." This is powerful motivation and pairs naturally with the streak badge.
Medium priority
Per-dimension detail view
Tapping a dimension in the Life Score breakdown could open a sheet showing exactly why the score is what it is — which items are dragging it down, with direct action buttons. Right now users see the score but can't easily trace where to go to improve it.
Future
AI-generated weekly summary
At weekly reset, the AI could generate a 2–3 sentence personalised summary: "You had a strong week on budget but meals planning slipped. The kids' routines were 80% complete. This week, focus on setting up next week's dinners." This would make the reset feel like a coach check-in rather than a form to fill.
Future
Household vs individual scoring
Currently the score is for the whole household. As multi-user support matures, it could split into personal accountability (routines, goals) vs shared household health (meals, maintenance, budget). The partner's contributions would show as their avatar's progress toward shared goals.
Future / Monetisation
Type A Mode as a paid tier
Type A Mode is currently free and always-on for anyone who enables it. Given its depth — Life Score, missions, weekly reset, streak — it has strong potential as a paid add-on or a feature unlocked at a higher subscription tier. The toggle in Settings is already isolated, making a paywall straightforward to add. Consider a free trial period (e.g. 2-week unlock) to drive conversion.
