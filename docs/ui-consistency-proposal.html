Neutral Grays — Replace Tailwind with Toto Warm Grays

Minor visual change

Onboarding and parts of Planner use Tailwind's cool slate grays. Toto uses warm beige-grays. This fix makes those sections feel warmer and more consistent — no section accent colours are changed.

Tailwind cool grays (current)
Container background #f8fafc
Label text #64748b
Dashed border #cbd5e1
Toto warm grays (proposed)
Container background --surface2
Label text --muted
Dashed border --hairline
Onboarding — Tailwind palette throughout onboarding.js:52, 156, 251, 367 Warm → warmer

Semantic States — Success / Warning / Danger

Minor colour shift

These are app-wide states (overdue, on-track, over-budget) that should look the same everywhere. Currently some places use Toto variables, others use hardcoded hex from different green/red families.

Current — two different greens in same app
#10b981 (Budget)
var(--good) (Routines)
Both mean "success" but are different greens
Proposed — one semantic green everywhere
var(--good) — #2EA86B
var(--good) — #2EA86B
Consistent across all sections
Budget surplus/deficit — hardcoded green/red budget.js:222, 230 Tiny colour shift
Before 
color:#10b981  /* surplus — emerald */
color:#ef4444  /* deficit — red-500 */
After 
color:var(--good)   /* #2EA86B — Toto green */
color:var(--alert)  /* #DC4F4F — Toto red */
#10b981 (emerald) is slightly brighter/cooler than --good (#2EA86B). Tiny shift — both read as "green". Teal wallet accent colour is untouched.
Planner urgency indicators — mixed hardcoded and vars planner.js:82–83 Minor
Before 
urgencyColor = overdue ? '#ef4444'
  : urgent ? 'var(--good)'
  : soon ? '#f59e0b'
  : '#94a3b8'
After 
urgencyColor = overdue ? 'var(--alert)'
  : urgent ? 'var(--good)'
  : soon ? 'var(--watch)'
  : 'var(--muted)'
Planner blue accent is untouched. Only the overdue/soon state indicators (red/amber) are standardised.
