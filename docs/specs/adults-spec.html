Adults

Design specification — a task, chore and prize system for adults. Mirrors the kids system but adds privacy, bidirectional assignment, and adult-appropriate UX. Completely separate data and UI from the Kids section.

01
Overview — What is this feature?
Why adults need their own task and reward system
▼

The Kids section already lets parents assign chores, set prizes, and track points for their children. The Adults section applies the same mechanic to the adults in the household — giving partners a fun, gamified way to assign tasks to each other and reward completion.

The core idea
Either adult can create a task and assign it to the other.
Either adult can create a prize and attach it to that task.
Tasks and prizes can be private — visible only to the assignee, not shared with the whole household.
The motivating example
Sam wants Chris to clean out the shed.
Sam creates the task and adds a private prize.
Chris sees the task and the prize — nobody else does.
Chris completes the task, Sam approves, Chris redeems.
💡 Why this is valuable: Household tasks between partners often carry friction — nagging, resentment, imbalance. A lighthearted points system with meaningful (and sometimes playful) rewards reframes chores as a fun exchange rather than a source of conflict. The privacy feature is essential: it allows adults to attach intimate or personal prizes without those showing up in a shared family view.

04
Privacy Model
Exactly who sees what, and when
▼

Privacy is the defining feature of the Adults section. The rules are simple and consistent — no exceptions.

Visibility rules

ViewerPrivate task/prizePublic task/prize
Assignee (person who has to do it)✅ Visible — it's theirs to action✅ Visible
Assignor (person who created it)✅ Visible — they created it✅ Visible
Any other adult🚫 Hidden completely✅ Visible
Any child🚫 Hidden — adults section not accessible🚫 Hidden

Privacy default

Tasks — default: Public
Most tasks are mundane household jobs. Defaulting to public means either adult can see what's outstanding without needing to opt in. The creator flips to private only when needed.
Prizes — default: Private
Prizes carry far more personal weight. Defaulting to private protects the creator's intent. They opt in to public only for innocuous prizes like "takeaway night" or "choose the movie".

Privacy in the UI

Private items are marked with a 🔒 Private badge visible only to the assignee and assignor. To every other user the item simply doesn't exist — no placeholder, no "hidden item" indicator, nothing that implies something is concealed.

⚠️ Storage caveat: Toto currently uses localStorage for persistence. This means all data — including private items — exists in the browser's local storage and could be read by a technically sophisticated user inspecting the device. For a production release, private items should be server-encrypted with per-user keys so that only the authorised member's session can decrypt them. This is a future-state requirement; the UI design is correct regardless.

⚠️ The notification content for private prizes shows only the prize name — no description. The full description is only visible inside the app to the assignee.

Rejection flow

If the assignor rejects a task completion, the task returns to "Outstanding" with an optional reason message. The assignee gets a notification: "Sam said: 'The shed floor wasn't swept — give it another go 😄'". Points are not awarded. The assignee can re-submit after re-doing the task.

09
End-to-End Walkthrough — The Shed Example
Sam, Chris, and the shed: full flow from creation to redemption
▼

This is the exact scenario from the brief, traced step by step through the system.

1
Sam
Creates the prize first
Sam opens Adults → I've Set → New Prize. Enters emoji 😏, name "Sexy time", point cost 200, assigns to Chris, adds a private description, and leaves privacy ON. Saves.
2
Sam
Creates the task, linked to the prize
Sam opens New Task. Enters emoji 🏚️, name "Clean out the shed", points 200, assigns to Chris, sets frequency to Once, and links to the "Sexy time" prize. Enables the private toggle. Saves.
3
System
Data written, privacy enforced
Both the task and the prize are stored with private: true, assignedTo: chris, assignedBy: sam. They do not appear in any other adult's view. If there were a third adult in the household, they would see nothing.
4
Chris
Logs in and sees the task
Chris opens Adults → My Tasks. He sees "🏚️ Clean out the shed" with a 🔒 Private badge and "+200 💜". The linked prize name is shown as a teaser: "Complete this to unlock 😏 Sexy time". Sam's prize description is not shown yet — only after redemption.
5
Chris
Does the task and marks it complete
Chris cleans the shed. In the app, he taps the ✓ on "Clean out the shed". The task moves to Pending Approval and turns amber.
6
System
Approval request sent to Sam
Sam receives a push notification: "Chris completed 'Clean out the shed' — approve to award 200 💜". The notification does not mention the linked prize.
7
Sam
Inspects and approves
Sam opens Adults → I've Set, sees the pending approval banner, taps Approve. 200 💜 are added to Chris's balance. The linked prize "😏 Sexy time" appears in Chris's Prize Store highlighted as "Just unlocked!"
8
Chris
Redeems the prize
Chris opens Prize Store, sees "😏 Sexy time" highlighted. He taps Redeem. A redemption request is sent to Sam. Chris's balance drops by 200 💜 (held pending approval, returned if rejected).
9
Sam
Approves the redemption
Sam receives a private push notification: "Chris wants to redeem 😏 Sexy time". Sam approves. The points are finalised. Chris gets a notification: "Sam approved — enjoy! 🎉". The full prize description is now shown to Chris inside the app.
✅ At no point did the task name "Clean out the shed", the prize name "Sexy time", or Sam's description appear in any shared household view, any child's session, or any notification visible to a third party. The entire exchange was private end-to-end.

11
Notifications
Push and in-app notification rules for private and public items
▼

Notifications are the riskiest surface for privacy leakage. A notification visible on a lock screen could expose a private task or prize name to anyone who sees the device. The rules below prevent this.

EventRecipientNotification text (private)Notification text (public)
Task created for me Assignee "Sam set you a new private task 🔒" "Sam set you a new task: Clean the car 🚗"
Task marked complete Assignor "Your partner completed a private task — approve it 💜" "Chris completed 'Service the car' — approve to award 50 💜"
Task approved Assignee "Task approved — 200 💜 added to your balance!" "Task approved — 50 💜 added! Keep it up 🎉"
Task rejected Assignee "Your task wasn't approved — open Toto for details" "Sam sent feedback on 'Service the car' — open Toto"
Prize redeemed Assignor "Chris wants to redeem a private prize 🔒 — open Toto" "Chris wants to redeem 'Pizza night' — approve?"
Redemption approved Assignee "Your prize was approved — enjoy! 🎉" "'Pizza night' approved — enjoy! 🍕"
🔒 Lock screen rule: Private notifications use generic text only. The actual task or prize name is never included in the notification payload for private items — it only appears inside the app after the correct user has authenticated. This holds even if the device is unlocked.
💡 Notification preference: Each adult should be able to toggle notifications for the Adults section independently — some people may prefer to check the app rather than receive push alerts for private content.
